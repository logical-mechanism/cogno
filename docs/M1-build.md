foundation) **+ a simplified write path** (post/delete with a simple-but-abstracted sr25519
signer). It maps onto `docs/L5-frontend.md` ¬ß12 milestones L5-M0 and a stripped L5-M1/M2.

- **IN:** Next.js static-export SPA (`output:'export'`) in `app/`; PAPI (polkadot-api) client
  over WS; live feed via `Posts.watchEntries()`; post/delete via an sr25519 `getPolkadotSigner`;
  in-block + finalized status (honest *signed ‚âÝ included*); the hand-authored CSS-variable

  endpoint-as-config; constants read from PAPI metadata.
- **DEFER (named, seams stubbed):** Cardano / MeshJS / CIP-8 / wallet identity (M2); talk-capacity

  keystore signer, so M2 slots in with no call-site change); the onboarding FSM; the indexer
  (PAPI-direct only); the anchor.

## Grounded PAPI 1.23.3 API surface (verified LIVE against spec v101 ‚Äî not guessed)

Confirmed by `app/scripts/papi-acceptance.mjs`, `watch-probe.mjs`, and a tx-watch probe:

- **Client:** `createClient(getWsProvider(ws))`; `getWsProvider` from `polkadot-api/ws-provider/web`
  (browser) or `‚Ä¶/node`. `client.getTypedApi(cogno)`. `client.finalizedBlock$` ‚Üí one
  `{hash,number,parent,hasNewRuntime}`; `client.bestBlocks$` ‚Üí array (tip = `[0]`); `client.destroy()`.
- **Constants:** `await api.constants.System.Version()` ‚Üí `{spec_name:"cogno-chain-runtime",
  spec_version:101, transaction_version:1, ‚Ä¶}`; `await api.constants.System.SS58Prefix()` ‚Üí **42**.
- **Write:** `api.tx.Microblog.post_message({ text: Binary.fromText(s), parent: undefined|bigint })`,
  `api.tx.Microblog.delete_post({ id: bigint })`. `.signSubmitAndWatch(signer)` emits, in order:
  `signed`/`broadcasted` `{type,txHash}` ‚Üí `txBestBlocksState {found,block:{number,hash},ok,events}`
  (`found:true` = in best block) ‚Üí `finalized {block,ok,events}`. `PostCreated`/`PostDeleted` ids
  live in `events` as `{type:"Microblog", value:{type:"PostCreated", value:{id,author}}}`.
  (`text` is `Vec<u8>` ‚Üí ALWAYS `Binary.fromText`; the @polkadot/api hex gotcha does NOT apply to PAPI.)
- **Read:** `Posts.getValue(id,{at})` ‚Üí `Post|undefined`; `Posts.getEntries({at})` ‚Üí
  `[{keyArgs:[bigint], value:Post}]`; `Posts.watchEntries()` ‚Üí `{block, entries, deltas}` where
  `entries` = full set `[{value:Post, args:[bigint id]}]` (authoritative; rebuild feed each emission)
  and `deltas` = `{deleted,upserted}` **or null** (don't rely on it). `NextPostId.getValue()` ‚Üí bigint;
  `ByAuthor.getValue(ss58)` ‚Üí `bigint[]|undefined`. `Post = {author:SS58, text:Binary(.asText()),
  parent?:bigint, at:number}`.
- **Signer:** `sr25519CreateDerive(entropyToMiniSecret(mnemonicToEntropy(DEV_PHRASE)))("//Alice")` ‚Üí
  `{publicKey, sign}`; `getPolkadotSigner(publicKey,"Sr25519",sign)`; `ss58Address(publicKey,42)`.
  Random session key via `generateMnemonic()` ‚Üí `mnemonicToMiniSecret`. The `AuthorizeCall` signed
  extension is derived from metadata automatically ‚Äî signing works end-to-end (verified).

### Gotcha resolved during grounding
`api.tx.X.Y.isCompatible(token)` is **wrong** ‚Äî `isCompatible(threshold, token?)` takes a
`CompatibilityLevel` first; passing the token as arg 1 makes it call `getCompatibilityLevel(undefined)`
and crash **asynchronously** (surfaces later, during an unrelated `await`). Don't pre-check
compatibility this way; descriptors generated from the live node are compatible by construction.
