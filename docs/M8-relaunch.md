# M8 — talk_vault contract relaunch (LIVE on preprod) + wallet UX

The L-01 audit fix (PR #4) rewrote `contracts/validators/talk_vault.ak` (pin
`reference_script == None` on the spend continuation). That moved the compiled blueprint
hash and therefore the `min_lock`-applied policy id / vault address, **orphaning the live
M7 vault**. M8 relaunches the contract under the new hash, re-proves the whole live loop,
and (frontend) adds a CIP-30 wallet + encrypted posting keystore + a full polish pass.

## 0. The hash move

| Layer | Old (M7) | New (M8) |
|---|---|---|
| Blueprint hash (`plutus.json`) | `cf0712dc…26c6ba62` | `49ffbfc6…6237aeac` |
| Applied policy id / vault hash (min_lock=100 ADA) | `a82d0ad6…446cacf9` | `168a9710…52489c7` |
| Vault address (owner skh `10133fde…`) | `addr_test1zz5z6zkk…6t6ca8` | `addr_test1zqtg49cs…svljrte` |
| Beacon name (= app-chain identity hash) | `287a99d2…0ae6be75` | **unchanged** (pure fn of owner) |

Because the beacon **name** is `blake2b_256(cbor.serialise(owner))` — independent of the
script hash — the app-chain identity hash and the `CognoGate` bind (`287a99d2… → poster`)
**survive the relaunch**; only the vault address + beacon **policy id** move. No re-bind.

The applied artifact is regenerated by the new committed generator:
`node contracts/scripts/regen-vault.mjs` → `contracts/vault.json`
(`applyParamsToScript(compiledCode, [{int:100000000}], "JSON")` + `resolveScriptHash`,
the live `paramForm` and the exact path the in-browser lock uses). `aiken check` green
(533 checks, 38/38).

## 2. Tooling added

- `contracts/scripts/regen-vault.mjs` — the previously-missing committed generator for
  `vault.json`. Run it after any `aiken build` that changes the validator, then redeploy.
- `services/committee/state.mjs` — read-only chain-state inspector (genesis / spec / bind /
  weight / post count), the companion to `op.mjs` (writes) and `sync-weight.mjs`.
